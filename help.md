# 📖 논문 리뷰어 사용 가이드 (Help Guide)

본 앱은 **영어 논문(PDF)을 깊게 파고들기 위한 최적의 워크플로우**를 제공합니다. `Immersive Translate` 확장을 사용 중이시라면 아래 절차대로 진행했을 때 가장 강력한 효과를 보실 수 있습니다.

---

## ❓ 자주 묻는 질문 (문서 준비 관련)

### Q1. 영어 PDF랑 한글 번역 PDF를 따로 넣어도 되나요?
**아니요.** 두 개의 파일을 따로 넣어 AI가 문장을 매칭하게 하는 방식은 PDF 레이아웃 파싱의 한계(줄바꿈 오류 등) 때문에 매칭 정확도가 매우 떨어지고 오류가 잦습니다.

가장 안정적이고 **토큰(비용)을 아끼는 방법**은 `Immersive Translate`가 이미 공짜로(혹은 저렴하게) 번역해둔 구조를 활용하는 것입니다. 제가 이번에 업데이트한 **[이머시브 한글 우선 모드]**가 바로 이 문제를 완벽하게 해결합니다.

### Q2. 한글 위주로 보되, 원문만 따로 확인하고 싶어요. (최신 업데이트!)
이제 앱 설정에서 **[Korean-First Reading]**을 켜시면 다음과 같이 작동합니다:
1.  **한글 우선**: 모든 문장이 기본적으로 한글로 나옵니다.
2.  **문장별 원문 확인**: 이해가 안 되는 한글 문장 위에 **마우스를 올리면** 즉시 영어 원문이 툴팁으로 나타납니다.
3.  **문장별 고정**: 문장을 **클릭**하면 해당 문장만 영어로 고정되어 대조하며 읽을 수 있습니다.
4.  **토큰 절약**: 이미 번역된 HTML을 사용하므로, 단순히 읽는 것만으로는 AI 비용이 전혀 들지 않습니다! (요약/설명 기능을 쓸 때만 비용 발생)

### Q3. 영어가 나오다가 중간에 한글이 섞여 나와요.
이것은 원본 HTML의 구조 문제일 가능성이 큽니다. 리더기는 최대한 문맥을 파악해 나누려고 노력하지만, 번역 도구가 문단을 쪼갠 방식에 따라 영향을 받습니다. `Immersive Translate` 설정에서 '문단 단위 번역'을 활성화한 후 다시 내보내 보세요.

## 🚀 단계별 워크플로우

### 1단계: 논문 준비 (Browser에서)
1.  브라우저(Chrome/Edge)에서 읽고 싶은 **영어 논문 PDF**를 엽니다.
2.  `Immersive Translate` 아이콘을 클릭하고 **[번역]**을 누릅니다.
3.  번역이 완료된 화면에서 우측 하단의 **[기타] -> [단일 HTML 파일로 내보내기]** (Bilingual HTML)를 선택합니다.

### 2단계: 리더기로 가져오기
1.  앱의 **[Library]** 패널에서 **[+]** 버튼을 누르거나 파일을 드래그합니다.
2.  잠시 기다리면 AI가 요약을 생성합니다.

### 3단계: 단축키 활용 (충돌 방지 업데이트)
크롬 브라우저 기본 단축키와의 충돌을 피하기 위해 **Alt** 키 조합으로 변경되었습니다.
- `Alt + H`: 하이라이트 (형광펜)
- `Alt + S`: AI 요약 (Summarize)
- `Alt + E`: AI 설명 (Explain)
- `Alt + Q`: AI 문답 (Question)
- `Alt + T`: 해당 문단의 번역 끄기/켜기
